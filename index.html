Protocol Position

AUMP is the user mandate layer for agentic systems.

MCP gives agents tools, A2A lets agents discover and message each other, and UCP/AP2 handle commerce and payment boundaries. AUMP defines what an agent is allowed to want, reveal, concede, commit, escalate, refuse, and prove for a user.

How It Composes

AUMP does not replace existing agent protocols.

MCP

Tool and context access

Agents call tools such as aump.evaluate_action before taking material steps.

A2A

Agent discovery and messages

Messages carry mandate metadata without leaking private mandate contents.

AUMP

User authority and evidence

The portable contract for preferences, limits, disclosure, escalation, and audit logs.

UCP/AP2

Commerce and payment boundary

Checkout-ready or payment-sensitive actions escalate instead of becoming silent commits.

Conformance

The contract SDKs must pass.

uv run aump-conformance validate fixtures

What conformance proves

Conformance proves that independent implementations agree on schema validation, lifecycle checks, policy decisions, bridge metadata, protected disclosure handling, budget limits, and escalation boundaries.

What it does not prove

It does not prove that a live model will always negotiate well. Agent behavior is proven by examples and production integrations that call the mandate evaluator before acting.
