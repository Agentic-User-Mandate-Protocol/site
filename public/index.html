Protocol Position

AUMP is the user mandate layer for agentic systems.

MCP gives agents tools, A2A lets agents discover and message each other, and UCP/AP2 handle commerce and payment boundaries. AUMP defines what an agent is allowed to want, reveal, concede, commit, escalate, refuse, and prove for a user.

How It Composes

AUMP does not replace existing agent protocols.

MCP

Tool and context access

Agents call tools such as aump.evaluate_action before taking material steps.

A2A

Agent discovery and messages

Messages carry mandate metadata without leaking private mandate contents.

AUMP

User authority and evidence

The portable contract for preferences, limits, disclosure, escalation, and audit logs.

UCP/AP2

Commerce and payment boundary

Checkout-ready or payment-sensitive actions escalate instead of becoming silent commits.

LLM Integration

Place AUMP in the agent loop before commitment.

plan -> evaluate_action -> act or escalate -> evidence
1

Load mandate

The agent session starts with a mandate ID, hash, version, budget, disclosure rules, and escalation policy.

2

Propose action

The LLM can draft tool calls, offers, messages, or checkout steps, but they become proposed actions first.

3

Evaluate

The runtime calls aump.evaluate_action before sending a material tool call or agent message.

4

Route decision

allowed proceeds, denied is blocked, and requires_escalation pauses for trusted review.

5

Bridge safely

MCP uses params._meta, A2A uses the extension URI, and UCP uses meta.aump.

6

Append evidence

Every material decision records action summary, mandate reference, downstream IDs, result, and reason codes.

Python guard

decision = runtime.evaluate_action(mandate_id, proposed_action)
if decision["decision"] == "allowed":
    result = await call_tool_or_agent(proposed_action)
elif decision["decision"] == "requires_escalation":
    result = await request_trusted_review(decision)
else:
    result = {"blocked": True, "reason_codes": decision["reason_codes"]}

runtime.append_evidence(
    mandate_id,
    "material_action_evaluated",
    proposed_action["summary"],
    decision["decision"],
    {"reason_codes": decision["reason_codes"]},
)

TypeScript guard

const decision = evaluateAction(mandate, proposedAction, { now });
if (decision.decision === "allowed") {
  await callToolOrAgent(proposedAction);
} else if (decision.decision === "requires_escalation") {
  await requestTrustedReview(decision);
} else {
  return { blocked: true, reason_codes: decision.reason_codes };
}

What conformance proves

Conformance proves that independent implementations agree on schema validation, lifecycle checks, policy decisions, bridge metadata, protected disclosure handling, budget limits, and escalation boundaries.

What it does not prove

It does not prove that a live model will always negotiate well. Agent behavior is proven by examples and production integrations that call the mandate evaluator before acting.
